# EPIC-UX-004: Patient 360 Profile (Generative Patient View)

**Priority:** P0 | **Estimated Effort:** 4 weeks | **Dependencies:** EPIC-UX-001, EPIC-UX-003
**Theme:** The Narrative Patient Experience - Moving from static tabs to dynamic storytelling

---

## Executive Summary

The Patient 360 Profile reimagines how healthcare providers interact with patient information. Instead of traditional tabbed interfaces with static data tables, we create a narrative, timeline-based view that tells the "Story of the Patient." This includes AI-powered summarization, conversational data queries, smart episode grouping, and contextual insights that surface the right information at the right time.

---

## Strategic Objectives

1. **Narrative View** - Replace tabs with a flowing timeline narrative
2. **Intelligent Grouping** - Auto-cluster events into clinical episodes
3. **Conversational Queries** - "Ask Patient Data" natural language interface
4. **Proactive Insights** - Surface care gaps, trends, and alerts
5. **Unified History** - Merge data from all channels (FHIR, Zoice, WhatsApp, App)

### Timeline Component
```typescript
interface TimelineProps {
```
